# --- deliberately NOT here: torch, transformers, scikit-learn -------------
# Listing them broke the Vercel deploy: the default index serves the CUDA
# build of torch, producing a 4,855 MB bundle against a 500 MB function
# limit. They are not needed to serve, only to run FinBERT sentiment and the
# offline evaluation, so they live in requirements-full.txt instead.
#
# What the serverless deployment loses: the Researcher agent reports
# UNAVAILABLE rather than a sentiment score. That is a real degradation and
# it is visible in every trace -- the fusion layer down-weights accordingly
# and the /ask UI says so, rather than quietly substituting a neutral value.
#
# For local development, the forward test, or a container with room:
#   pip install -r requirements-full.txt
#
